### Phase 5: 기존 추출 DB 엑셀 병합 및 지정 폴더 개별 저장 개편
- [x] **기존 DB 엑셀 불러오기 (Local History Merge)**
  - 대상 파일: `ui_daum.py`, `ui_jobkorea.py`, `history_manager.py`
  - UI 추가: 하단 컨트롤 영역에 `기존 추출 DB 불러오기` 버튼 추가.
  - 파일 파싱 로직: 기존에 수집했던 엑셀 파일을 선택하면 `pandas`를 이용해 이메일, 도메인(또는 홈페이지) 데이터를 일괄 추출.
  - 동기화 및 충돌 방지: 추출된 데이터를 `history_manager.add_record()`를 통해 로컬 `local_history.json`에만 대량 업데이트. (구글 시트의 클라우드 BlockList와는 완벽히 분리된 로컬 파일이므로 충돌 없음).
- [x] **키워드별 지정 폴더 개별 엑셀 자동 저장**
  - 대상 파일: `ui_daum.py`, `ui_jobkorea.py`
  - 저장 경로 로직 변경: 임의의 autosave 폴더가 아닌, 사용자가 UI의 `📁 저장폴더 선택` 버튼으로 미리 지정해 둔 **[사용자 지정 폴더]** 경로를 활용.
  - 수집 엔진 변경: 다중 키워드 큐(Queue) 진행 중, 1개의 키워드 수집이 완료될 때마다 즉시 `[지정된_경로]/플랫폼_키워드명_결과.xlsx` 형태로 단일 독립 엑셀 파일로 내보내기.
  - 데이터 격리: 저장 직후 UI 테이블 및 백그라운드 `data_list`를 완벽히 초기화(Clear)하여 다음 키워드 수집 결과에 이전 데이터가 섞이지 않도록 처리.
